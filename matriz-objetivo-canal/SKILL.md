# Matriz Objetivo x Canal Movistar

Cargada por el Planner/Planner y el Campaign Manager. Referencia estructurada para decidir que canales activar segun el objetivo de la campana.

Estas matrices son punto de partida. El Planner puede ajustarlas si el briefing lo justifica, pero debe documentar la desviacion.

---

## 1. Objetivo: Notoriedad

| Canal | Recomendacion | Rol |
|---|---|---|
| Medios masivos | Principal | Construir memoria, superioridad y territorio |
| Digital video / social reach | Principal | Amplificar cobertura y codigos de campana |
| Programatica premium | Apoyo | Cobertura cualificada y contexto |
| Movistar+ lineal | Apoyo | Refuerzo audiovisual en entorno propio |
| CRM | No principal | Solo si hay mensaje de valor especifico para clientes |
| Tienda | Apoyo | Presencia visual coherente, no presion comercial |
| Performance | No recomendado como protagonista | Puede capturar demanda derivada |

**Correccion tipica:** no se recomienda activar CRM masivo para un objetivo puro de notoriedad salvo que exista un valor concreto para clientes. En caso contrario, se convierte una campana de marca en presion comercial.

---

## 2. Objetivo: Consideracion

| Canal | Recomendacion | Rol |
|---|---|---|
| Digital social / Meta | Principal | Explicar beneficios y razones para elegir |
| Display secuencial | Apoyo | Ordenar mensajes por fases |
| Programatica contextual | Apoyo | Asociar producto a contexto relevante |
| Movistar+ banner / lineal | Apoyo fuerte | Refuerzo contextual en clientes |
| CRM | Principal si es base cliente | Personalizar razon de valor |
| Tienda | Principal si hay decision asistida | Explicar y reducir dudas |
| Performance | Apoyo | Capturar senales de intencion |

**Correccion tipica:** se recomienda usar Meta no solo como performance, sino como consideracion visual. Si solo se activa con precio, se pierde construccion de valor.

---

## 3. Objetivo: Performance / conversion

| Canal | Recomendacion | Rol |
|---|---|---|
| Meta performance | Principal | Capturar demanda con creatividad directa |
| Search / SEM | Principal | Capturar intencion activa |
| Retargeting | Principal con limites | Recordar, resolver objecion y cerrar |
| CRM | Principal si cliente identificable | Activacion personalizada |
| Tienda | Principal si hay cierre asistido | Conversion con explicacion humana |
| Movistar+ banner | Apoyo | Recordatorio contextual |
| Movistar+ preroll | Condicionado | Solo si la relevancia justifica interrupcion |
| First Impression | Normalmente no recomendado | Reservado para mensajes estrategicos |
| Medios masivos | Apoyo indirecto | Generar demanda, no optimizar conversion directa |

**Correccion tipica:** no se recomienda usar First Impression para una campana puramente tactica de conversion. El formato es demasiado valioso e intrusivo para un mensaje de bajo valor estrategico.

---

## 4. Objetivo: Fidelizacion / valor cliente

| Canal | Recomendacion | Rol |
|---|---|---|
| CRM | Principal | Reconocer, personalizar y activar valor |
| Movistar+ banner | Principal / apoyo fuerte | Recordatorio contextual de valor |
| Tienda | Apoyo | Explicar ventajas y resolver dudas |
| Digital customer match | Apoyo | Refuerzo segmentado |
| Movistar+ preroll | Condicionado | Solo para beneficios de alto valor |
| Medios masivos | Apoyo | Construir orgullo de pertenencia |
| Performance abierto | No recomendado | Puede parecer captacion generica |

**Correccion tipica:** no conviene tratar una campana de fidelizacion como una campana de descuento. El beneficio debe sentirse como reconocimiento, no como liquidacion.

---

## 5. Objetivo: Educacion / explicacion

| Canal | Recomendacion | Rol |
|---|---|---|
| Tienda | Principal | Explicar con ayuda humana |
| CRM | Principal | Ordenar informacion de forma clara |
| Digital video corto | Apoyo | Demostracion simple |
| Landing / web | Principal | Profundizar y resolver dudas |
| Movistar+ banner | Apoyo | Recordatorio sencillo |
| Movistar+ preroll | Condicionado | Solo si es breve y contextual |
| Performance | Apoyo bajo | Llevar a landing explicativa |
| Medios masivos | No recomendado como explicativo | Puede abrir territorio, no explicar detalle |

**Correccion tipica:** no se recomienda resolver una explicacion compleja en un banner o preroll. Estos canales deben abrir la puerta; la explicacion debe vivir en CRM, tienda, landing o contenido especifico.

---

## 6. Objetivo: Retencion

| Canal | Recomendacion | Rol |
|---|---|---|
| CRM | Principal | Mensaje personalizado y reconocimiento |
| Call center | Principal si hay riesgo alto | Resolucion, escucha y cierre |
| App / web privada | Apoyo | Autogestion y consulta |
| Tienda | Apoyo si hay visita | Explicacion y recuperacion |
| Movistar+ banner | Condicionado | Refuerzo suave de valor |
| Digital retargeting | Muy condicionado | Evitar sensacion de persecucion |
| Masivos | Apoyo indirecto | Reforzar confianza de marca |

**Correccion tipica:** en retencion, la presion comercial debe sentirse como cuidado, no como desesperacion. No se recomienda usar retargeting agresivo si el cliente ya esta recibiendo CRM o contacto de servicio.

---

## Como usar estas matrices

1. **Planner/B**: identifica el objetivo principal del briefing, consulta la matriz correspondiente. Si el briefing pide canales que la matriz marca como "no recomendado", flaggea la discrepancia.
2. **Campaign Manager**: en Cierre, verifica que la arquitectura de canales es coherente con el objetivo. Si una campana de notoriedad tiene CRM como canal principal sin justificacion, es un flag.
